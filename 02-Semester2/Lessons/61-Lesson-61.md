## 2. النظري (10 دقائق)

### لماذا مراجعة الأقران؟

مراجعة الأقران (Peer Review) هي **أداة تعليمية قوية** يستخدمها الموسيقيون والمبدعون في كل المجالات.

```diff
+ الفائدة: ترى أخطاء لم تكن تراها، تتعلم من حلول الآخرين، تحسن عملك
+ للطالب المراجع: تتعلم كيف تنقد وتقيّم — مهارة حياتية مهمة
+ للطالب صاحب العمل: تحصل على رأي جديد خارج منظورك الشخصي
```

### مبادئ النقد البناء (Constructive Criticism)

1. **ابدأ بالإيجابي:** أخبر زميلك بما يعجبك في عمله أولاً
2. **كن محدداً:** لا تقل "هذا خطأ" — قل "هذه النوتة قد تكون خارج الميزان"
3. **اقترح حلولاً:** لا تذكر المشكلة فقط — قدم اقتراحاً للتحسين
4. **كن محترماً:** تذكر — هناك إنسان وراء هذا العمل
5. **ركّز على العمل وليس الشخص:** "الهارموني هنا غير متناسق" — ليس "أنت لم تفهم الهارموني"

### قواعد الجلسة

| القاعدة | الشرح |
|---------|-------|
| 🔇 استمع أولاً | اسمع العمل كاملاً قبل أن تنقد |
| 📝 دوّن ملاحظات | اكتب ما لاحظته — لا تعتمد على الذاكرة فقط |
| ⏱ احترم الوقت | لكل مشروع وقت محدد للمراجعة |
| 🤝 كن عادلاً | عامل عمل زميلك كما تحب أن يعاملوا عملك |
| 💡 قدّم قيمة | اسأل نفسك: "هل هذا التعليق سيساعد زميل؟" |

## 3. التطبيق العملي (30 دقيقة)

### خطوة 1: تشكيل أزواج المراجعة

1. سيقوم المدرس بتقسيم الطلاب إلى **أزواج (Pairs)** أو **مجموعات صغيرة (3 طلاب)**
2. سيحصل كل طالب على **زميل مراجع (Reviewer)**
3. سيكون لديك **15 دقيقة لكل مشروع** (مراجعة + مناقشة)

### خطوة 2: قائمة المراجعة (Peer Review Checklist)

استخدم هذه القائمة لتقييم عمل زميلك:

**قسم 1: المحتوى الموسيقي والتقني**

| البند | نعم | لا | ملاحظات |
|-------|-----|----|---------|
| هل هناك عنوان واضح للمقطوعة؟ | | | |
| هل الميزان مناسب ومضبوط؟ | | | |
| هل المقام/المفتاح الموسيقي مضبوط؟ | | | |
| هل كل المقاييس تحتوي على العدد الصحيح من النبضات؟ | | | |
| هل النوتات ضمن النطاق المناسب للآلة؟ | | | |
| هل هناك تنوع في القيم الإيقاعية؟ | | | |
| هل التجميع (Beaming) صحيح؟ | | | |
| هل هناك علامات عرضية صحيحة؟ | | | |

**قسم 2: الهارموني والمرافقة**

| البند | نعم | لا | ملاحظات |
|-------|-----|----|---------|
| هل الكوردات مناسبة للّحن؟ | | | |
| هل هناك تنوع في تتابع الكوردات؟ | | | |
| هل المرافقة تدعم اللحن دون طغيان؟ | | | |
| هل خط الباس متنوع ومنطقي؟ | | | |

**قسم 3: التنسيق والعرض**

| البند | نعم | لا | ملاحظات |
|-------|-----|----|---------|
| هل توزيع المقاييس على الصفحة متوازن؟ | | | |
| هل الهوامش مناسبة؟ | | | |
| هل النوتة مقروءة وواضحة؟ | | | |
| هل هناك ديناميكيات (p, mf, f)؟ | | | |
| هل هناك وسائل أداء أخرى؟ | | | |

**قسم 4: الصوت والمكسير**

| البند | نعم | لا | ملاحظات |
|-------|-----|----|---------|
| هل مستويات الصوت متوازنة؟ | | | |
| هل هناك تحريك (Pan) مناسب؟ | | | |
| هل الصوت العام واضح ونقي؟ | | | |

### خطوة 3: عملية المراجعة — لكل زميل 15 دقيقة

**الجدول الزمني (30 دقيقة = 2 زميل × 15 دقيقة):**

| الوقت | النشاط |
|-------|--------|
| **0-2 دقيقة** | شغّل المشروع واستمع كاملاً (بدون كلام) |
| **2-5 دقائق** | املأ قائمة المراجعة — اكتب ملاحظاتك |
| **5-10 دقائق** | ناقش ملاحظاتك مع زميلك — ابدأ بالإيجابي |
| **10-12 دقيقة** | شغّل المقاطع التي أشرت إليها واشرح ملاحظاتك |
| **12-15 دقيقة** | زميلك يطرح أسئلة ويطلب توضيحات |

### خطوة 4: تقديم التغذية الراجعة — نموذج الحوار

**ابدأ هكذا:**
> "أعجبني جداً لحنك في المقاييس الأولى — له طابع جميل. لاحظت أن في المقياس 7، مجموع النبضات أقل من 4..."

**استخدم "أنا" بدلاً من "أنت":**
- ✅ "أشعر أن الهارموني في المقياس 12 غير مستقر"
- ❌ "الهارموني عندك خطأ في المقياس 12"

**اقترح تحسينات:**
- ✅ "ماذا لو جربت كورد Dm بدلاً من F في المقياس 8؟ قد يعطي شعوراً مختلفاً"
- ❌ "الكورد في المقياس 8 غلط"

**أنهِ بإيجابية:**
> "بشكل عام، عمل رائع! مع بعض التعديلات البسيطة، سيكون المشروع ممتازاً"

### خطوة 5: استلام التغذية الراجعة — كيف تتصرف

عندما يراجع زميلك مشروعك:

1. **استمع بانفتاح:** لا تدافع عن عملك — اسمع أولاً
2. **دوّن الملاحظات:** اكتب كل تعليق يقوله زميلك
3. **اسأل أسئلة توضيحية:**
   - "هل يمكنك توضيح ما تقصد؟"
   - "أين بالضبط لاحظت المشكلة؟"
   - "هل لديك اقتراح بديل؟"
4. **لا تأخذ النقد بشكل شخصي:** المراجعة تركز على العمل وليس عليك
5. **اشكر زميلك:** "شكراً لهذه الملاحظات — ستساعدني كثيراً"

### خطوة 6: أولويات التعديل — ما يجب تغييره أولاً؟

بعد استلام التغذية الراجعة، رتّب التعديلات حسب الأولوية:

| الأولوية | نوع التعديل | مثال |
|---------|-------------|------|
| 🔴 عاجل | أخطاء إيقاعية أو نوتية | مقياس ناقص نبضات |
| 🟡 مهم | تحسين الهارموني | كورد غير مناسب |
| 🟢 تحسيني | تنسيق وعرض | هوامش غير متساوية |
| ⚪ اختياري | ذوق شخصي | تغيير نمط المرافقة |

### خطوة 7: تطبيق التعديلات على مشروعك

1. افتح ملفك (كل طالب يعمل على ملفه الآن)
2. ابدأ بالتعديلات **العاجلة (🔴)** أولاً
3. ثم التعديلات **المهمة (🟡)**
4. ثم **التحسينية (🟢)** إذا بقي وقت
5. احفظ الملف بعد كل تعديل مهم: **Ctrl+S**

### خطوة 8: التعديلات المقترحة الشائعة

**تصحيح الأخطاء الإيقاعية:**
- إذا كان المقياس ناقص نبضات: أضف نوتة أو سكتة
- إذا كان زائد: احذف نوتة أو استخدم قيمة أطول

**تحسين الهارموني:**
- إذا كان الكورد لا يتوافق مع اللحن: جرب كورداً بديلاً
- إذا كانت الانتقالات متكررة: أضف كورداً متنوعاً

**تحسين التوزيع:**
- إذا كان هناك مقياس مزدحم: أضف System Break
- إذا كان النص صغيراً: كبّر حجم المدرج

## 5. مراجعة وواجب (5 دقائق)

### أسئلة المراجعة

1. **لماذا نبدأ النقد البناء بالإيجابي؟** — لخلق جو من الثقة والاحترام قبل تقديم الملاحظات الناقدة
2. **ماذا تفعل إذا اختلفت مع ملاحظة زميلك؟** — اسأل عن التفاصيل، اشرح وجهة نظرك، قد تتفقان على حل وسط
3. **ما الفرق بين النقد البناء والنقد السلبي؟** — النقد البناء يقدم ملاحظات محددة مع اقتراحات للتحسين
4. **لماذا من المهم أن يراجع شخص آخر عملك؟** — لأنك قد تكون أعمى عن أخطائك، والآخر يرى ما لا تراه

### الواجب المنزلي

1. طبّق على الأقل **3 تعديلات** من تغذية زميلك الراجعة على مشروعك
2. اكتب قائمة بالتعديلات التي قمت بها وسبب كل تعديل
3. احفظ الملف بعد التعديلات باسم: `Lesson61-Reviewed.sib`
4. اكتب جملة تصف شعورك بعد تطبيق التعديلات — هل العمل أفضل الآن؟
5. استعد لمناقشة التعديلات في الدرس القادم

## 6. تطبيق عملي

- **على آلتك الموسيقية:**
  - اعزف المقطع الذي قمت بتعديله — هل يبدو أفضل الآن؟
- **في Sibelius:**
  - قارن بين النسخة الأصلية والنسخة المعدّلة — استمع للفرق
- **تحدي إضافي:**
  - أعد تقييم عملك باستخدام نفس قائمة المراجعة بعد أسبوع — منظور جديد!

## 9. ملاحظات المدرس

- **هذا الدرس تفاعلي — شجّع الحوار المفتوح والاحترام بين الطلاب**
- **للطلاب الجدد:** قد يكون النقد صعباً — ذكّرهم أن الهدف هو التحسن وليس النقد الشخصي
- **للطلاب المتقدمين:** اطلب منهم تقديم اقتراحات إبداعية وليس فقط تصحيح أخطاء
- **خطأ شائع:** تحويل النقد البناء إلى نقد جارح — تدخل فوراً إذا حدث هذا
- **نصيحة:** إذا كان هناك طالب خجول، اجلس معه وساعده في صياغة ملاحظاته
- **اقتراح نشاط إضافي:** بعد المراجعة، اطلب من كل طالب أن يقول شيئاً واحداً تعلمه من مشروع زميله
- **ربط مع الواقع:** في صناعة الموسيقى، الملحنون يعملون مع محررين (Editors) يراجعون أعمالهم — هذه هي البداية
- **آلتك الموسيقية:** كل نقد تقدمه اليوم سيساعد زميلك غداً، وكل نقد تستلمه سيساعدك أنت. نحن ننمو معاً!
